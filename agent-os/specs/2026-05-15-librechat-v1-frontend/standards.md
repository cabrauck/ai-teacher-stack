# Standards for LibreChat v1 Teacher Frontend

- `security-compliance`: local-first defaults, no secrets in repo, no duplicate
  cloud or RAG paths by default, auditable release boundary.
- `privacy-boundary`: no student names, grades, diagnoses, parent
  communication, health data, performance records, credentials, or confidential
  school documents in defaults, docs, examples, tests, or memory.
- `contributor-workflow`: Agent-OS remains the spec gate; contributor tools are
  separate from the teacher product frontend.
- `document-output`: Markdown and DOCX exports stay first-class outputs.
